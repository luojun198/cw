## [2026-06-09] FIX-022 物料档案页面空白且数据无法加载（两处 JS 错误）

- **严重级别**：P1（核心页面 `/scm/items` 整页空白、无法使用）
- **问题来源**：用户反馈"http://localhost:5175/scm/items 总是显示空白无法加载"
- **影响范围**：`client/src/views/scm/ItemList.vue`（仅前端）
- **根因**（两个独立 bug 叠加）：
  1. **整页崩溃（TDZ）**：新增自定义字段列功能时，`activeFieldDefs` 声明在文件约 970 行，但被约 519 行的 `allColumnSettings` 计算属性及约 552 行 `watch(allColumnSettings, …, { immediate: true })` 引用。immediate watch 在 setup 阶段同步求值时 `activeFieldDefs` 尚处暂时性死区（TDZ），setup 抛错导致 `$setup` 部分值变 `undefined`，模板中 `dynamicItemTypes[row.item_type]`（item_type 为 null）退化为 `undefined['null']` → `TypeError: Cannot read properties of undefined (reading 'null')`，cell 渲染崩溃、整页空白。

- **修复**：
  1. 将 `const activeFieldDefs = ref([])` 提前到 `allColumnSettings` 计算属性之前声明，消除 TDZ。
  2. 删除 `onMounted` 中无效的 `loadColumnConfig()` 调用（列可见性初始化已由 `allColumnSettings` 的 immediate watch 处理）。
- **验证**：重启前端后访问 `/scm/items`，页面正常渲染；自动加载到 1 个物料"01 测试物料"，表头含自定义字段列"产地"，`activeFieldDefs` 加载到 1 个字段；无控制台报错。前端改动，无需重启后端。
- **回滚方法**：还原 `activeFieldDefs` 声明位置并在 `onMounted` 恢复 `loadColumnConfig()` 调用即可复现（不建议）。

## [2026-06-08] FIX-018 供应链单据审核/凭证的三处账务缺陷

- **严重级别**：P0（生成的凭证借贷不平衡，无法过账；发票应收应付完全丢失；调拨/盘点静默空操作）
- **问题来源**：会话内对供应链（SCM）P2–P5 已写未联调代码的逐函数审查
- **影响范围**：`server/src/routes/scmDoc.ts`（单据审核/反审核、生成凭证、应收应付账龄）、`server/src/services/scmArAp.ts`（往来台账）
- **修改文件**：
  - `server/src/routes/scmDoc.ts`
  - `server/src/services/scmArAp.ts`

### 缺陷 1：`gen-voucher` 生成的凭证借贷不平衡
- **根因**：原 `POST /scm/docs/:id/gen-voucher` 只按单据方向生成**单边分录**——入库仅借存货、出库仅贷销售科目，没有对方科目，借≠贷，凭证无法过账；且 `total_amount` 误用 `借+贷` 之和。
- **修复**：按单据类别（purchase/sale/production/inventory）+ 方向生成完整且配平的分录：
  - 采购入库：借 存货 + 进项税 / 贷 应付；采购退货反向。
  - 销售出库：确认收入（借应收 / 贷收入 + 销项）+ 结转成本（借成本 / 贷存货，成本取审核回写的 `unit_cost`）；销售退货反向。
  - 完工入库 / 生产领用：存货 ↔ 生产成本对开。
  - 其他出入库：存货 ↔ 待处理财产损溢对开。
  - 对方科目优先取往来单位的应收/应付科目，其次取系统参数（`scm:acc_ap/ar/revenue/cost/other/production/tax_input/tax_output`），最后用标准科目兜底（2202/1122/6001/6401/1901/5001/22210101/22210105）。
  - 缺科目时聚合报错列出缺失编码；生成前做借贷平衡校验（容差 0.01），`total_amount` 改为借方合计。

### 缺陷 2：采购/销售发票（RP/RS）审核不生成应收应付
- **根因**：`scmArAp.ts` 注释声明发票审核生成应收应付台账，但 `audit` 仅对 PAY/RCV 调 `applyPayment`，RP/RS（`affects_ar_ap=1`）审核时无任何动作，往来台账缺失发票基数。同时发现 `getPartnerLedger`（in 增/out 减）与 `arap-aging`（in 减/out 增）对同一 `direction` 的余额符号相反。
- **修复**：
  - 新增 `applyInvoice()`：RP/RS 审核时按价税合计写入 `scm_ar_ap_log`（`direction='in'`，不产生出纳流水）；`audit`/`unaudit` 接入（反审核复用 `reversePayment` 按 `doc_no` 清台账）。
  - 统一往来台账符号约定：**`in`=往来余额增加（发票）、`out`=减少（收付款）**。`applyPayment` 的收付款 `direction` 统一为 `out`；修正 `arap-aging` 符号与 `getPartnerLedger` 一致。

### 缺陷 3：`direction='none'` 单据审核为静默空操作
- **根因**：调拨（TR）/盘点（CK）/组装（AS）/拆卸（DS）方向为 none，审核循环两个分支都不命中，库存零变动却把状态改为 `audited`，用户误以为生效。
- **修复**：
  - 调拨 TR：约定 单据仓库=调出仓、明细行仓库=调入仓；审核时源仓 `applyMovingAvgOut` 取移动均价、按该成本 `applyMovingAvgIn` 入调入仓；前置校验调入仓非空、≠源仓、源仓库存充足。
  - 盘点 CK：明细行 `qty`=实盘数，与账面比较，盘盈按账面均价入、盘亏出。
  - 组装/拆卸 AS/DS：依赖 BOM 配料流程尚未实现，审核时明确返回 400 拒绝，杜绝静默空操作。
  - 反审核：TR 两条移动（源仓出/目标仓入）、CK 盈亏移动均由既有 `reverseDocMoves` 按 `doc_type-doc_no` 正确回退。
- **新增/删除字段或表**：无（复用现有表结构；TR 调入仓借用明细行 `warehouse_code`，CK 实盘数借用明细行 `qty`）
- **数据修复**：无（SCM 为新模块，仅测试样本数据）
- **测试**：`npx tsc --noEmit -p server/tsconfig.json` 对改动文件无类型错误；后端 `--watch` 自动重载。端到端联调（采购入库→销售出库→收付款→生成凭证→调拨/盘点）待下一阶段在登录态下验证。
- **风险与回滚**：纯逻辑修复，不涉及迁移。回滚还原 `scmDoc.ts`、`scmArAp.ts` 即可。
- **相关说明**：详见 `优化日志.md` 同日「供应链模块缺陷修复」条目。

  - `client/src/views/asset/AssetList.vue`：`onMounted` 的 `getDicts()` 与 `handleQuery()` 改为并行（二者无依赖）。
