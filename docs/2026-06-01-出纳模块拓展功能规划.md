## 二、拓展功能规划（按优先级分 P0–P3）

### P0：银行余额调节表（核心报表，企业必备）

**业务描述**：月末/期末，企业银行存款日记账余额与银行对账单余额因未达账项（在途资金、未兑现支票等）不一致，需要编制余额调节表找出差异原因。

**技术方案**：
1. **后端**：`GET /api/cashier/reconciliation?account_code=&end_date=`
   - 读取 `cashier_journal` 企业账余额（期初 + 累计借 − 累计贷）
   - 读取 `bank_statement` 银行账余额
   - 统计未达账项：企业已记银行未记 / 银行已记企业未记（即 `reconciled=0` 或 `matched=0`）
   - 计算调节后余额（双方应一致）
2. **前端**：`views/cashier/BankReconciliation.vue`
   - 上部：「企业账面余额」vs「银行对账单余额」卡片
   - 中部：未达账项表（企业已记银行未记 / 银行已记企业未记，两栏对照）
   - 下部：调节后余额（绿色=一致，红色=不一致并计算差额）
   - 支持打印和导出 Excel

**DDL 影响**：无，复用现有 `cashier_journal` + `bank_statement`。

---

### P1：凭证与出纳日记账自动同步

**业务描述**：录入凭证时若涉及现金（1001）或银行存款（1002）科目，自动生成对应出纳日记账记录。

**技术方案**：
1. **钩子**：在 `voucherEntry.ts` 的 `createVoucher` / `renameAndSave` 完成后，检测分录是否包含 `is_cash=1` 或 `is_bank=1` 的科目
2. **自动生成**：对每个涉及现金/银行的凭证分录，写入 `cashier_journal`：
   - `account_code` = 科目编码
   - `debit/credit` = 分录金额
   - `voucher_year/month/type/no` = 凭证回链
   - `summary` = 凭证摘要
3. **双向追溯**：
   - 日记账列表已含凭证回链显示（现有）
   - 凭证详情页增加「出纳日记账」联动入口（跳转或侧边栏）
4. **开关**：在 `system_params` 增加 `cashier:auto_sync_from_voucher`（默认 false），允许按账套独立开关

**DDL 影响**：在 `system_params` 插入一条配置记录。

---

### P2：银行对账单 Excel 导入

**业务描述**：银行提供的对账单通常为 Excel/CSV 格式，需要一行一行手动录入效率极低。

**技术方案**：
1. **上传 API**：`POST /api/cashier/bank-statement/import`（multipart/form-data）
   - 接收 .xlsx/.xls/.csv
   - 使用 `xlsx` 库解析
2. **列映射 UI**（类似 Airtable 导入）：
   - 拖拽或下拉指定列映射：日期→biz_date / 借方→debit / 贷方→credit / 票据号→bill_no / ...
   - 支持跳过前 N 行、识别表头行、排除汇总行
3. **预览 + 确认**：解析后显示前 10 行，确认无误后批量写入 `bank_statement`
4. **预存映射**：按银行账号保存列映射配置，下次同账号导入无需重复配置

**DDL 影响**：新增 `bank_import_profile` 表（存储列映射配置）；依赖 `npm install xlsx`。

---

### P3：出纳日报 / 资金日报

**业务描述**：出纳人员每天需要编制资金日报，汇总当日现金/银行收付情况及余额变动。

**技术方案**：
1. **API**：`GET /api/cashier/daily-report?date=`
   - 按科目汇总当日借方（收入）、贷方（支出）
   - 返回每科目：期初余额、本期收入、本期支出、期末余额
   - 合计行：全部现金/银行汇总
2. **前端**：`views/cashier/DailyReport.vue`
   - 日期选择器（默认当天）
   - 汇总表格：科目 | 期初 | 收入 | 支出 | 期末
   - 合计行高亮
   - 支持打印和导出

**可选拓展**：月报（汇总整月数据，用 `cn_hz` 结果）。

**本期策略**：

- **不入前端菜单**：不暴露给用户
- **后期判定**：如果实际用户 ACD 文件中有票据数据，再单独规划进销存模块

## 三、实施建议

### 推荐路径（按优先级顺序）

| 阶段 | 内容 | 预估工作量 | 依赖 |
|------|------|-----------|------|
| **E1** 银行余额调节表 | M3-P0 | 中等（1 后端 + 1 页面） | 无 |
| **E2** 凭证自动同步 | M3-P1 | 小（钩子 + 配置开关） | 无 |
| **E3** 银行对账单导入 | M3-P2 | 大（上传 + 列映射 UI + 预览） | `xlsx` 库 |
| **E4** 出纳日报 | M3-P3 | 小（1 API + 1 页面） | 无 |

### 推荐先做 E1 + E2（余额调节表 + 凭证同步）

理由：
1. **余额调节表**是出纳最核心的月末报表，财务审计必备
2. **凭证自动同步**打通"凭证→日记账"数据流，解决手动维护痛点
3. 两者都不依赖外部库，改动范围小、收益大
4. E3（Excel 导入）需新增依赖和较重的 UI 交互，可独立排期

## 四、技术注意事项

1. **余额调节表精度**：企业账和银行账余额计算必须用完全相同的区间，特别是期初余额取数逻辑与日记账列表的 `listJournal` 保持一致
2. **凭证同步幂等**：同一凭证重复提交不应产生重复日记账（按 `voucher_year/month/type/no + account_code` 唯一约束）
3. **Excel 导入性能**：大银行流水文件（10万+行）需分批写入 + 进度回调，复用现有 `taskQueue`
4. **前端复用**：日报/月报可共用相同的汇总表格组件，E4 可与 E1 的调节表共用一个导出/打印工具栏
