# 固定资产 与 出纳 模块开发计划

> 编制日期：2026-06-01
> 目标：在 CW Finance 中新增「固定资产」「出纳管理」两大模块，**完全兼容**润衡 `.acd` 备份文件中相关表的导入与还原。
> 依据：标准模版 `标准模版/新企业会计准则/新企业会计准则.acd`（通过 `rh备份读取工具/acd_reader.py` 解析得到的 `data_stru.txt` 建表语句）。

---

## 一、ACD 源表结构总览（兼容基线）

ACD 数据以 GBK 编码、`\t` 分隔的文本存储于 `rhsj\*.txt`，建表语句在 `data_stru.txt`。本计划必须保证导入时**字段不丢失、可逆还原**。

## 二、目标数据库设计（SQLite，新增表）

遵循现有工程命名规范（蛇形、`account_set_id` 多账套隔离、统一审计字段）。每张表附带 `acd_*` 兼容映射列以实现可逆还原。

### 2.1 出纳模块表

```sql
-- 出纳日记账（对应 cn_mx）
CREATE TABLE cashier_journal (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id INTEGER NOT NULL,
  account_code TEXT NOT NULL,        -- kmbm 现金/银行科目
  currency TEXT DEFAULT 'RMB',       -- bz
  biz_date TEXT NOT NULL,            -- rq
  seq INTEGER,                       -- xh
  summary TEXT,                      -- zy
  debit REAL DEFAULT 0,              -- jf
  credit REAL DEFAULT 0,             -- df
  settle_type TEXT,                  -- 结算方式 jslb
  bill_no TEXT,                      -- 票据号
  counter_unit TEXT,                 -- g_qymc 对方单位
  counter_account TEXT,              -- dfkm 对方科目
  bank_name TEXT,                    -- khyh
  bank_account TEXT,                 -- yhzh
  unit_code TEXT,                    -- dwbm
  reconciled INTEGER DEFAULT 0,      -- gzm/对账标志
  voucher_year INTEGER, voucher_month INTEGER,
  voucher_type INTEGER, voucher_no INTEGER, -- p_nf/p_yf/p_pzlx/p_pzbh
  created_by TEXT, created_at TEXT, updated_at TEXT,
  FOREIGN KEY(account_set_id) REFERENCES account_sets(id)
);

-- 出纳期初余额（对应 cn_nc）
CREATE TABLE cashier_init_balance (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id INTEGER NOT NULL,
  account_code TEXT NOT NULL, currency TEXT DEFAULT 'RMB',
  balance REAL DEFAULT 0,
  UNIQUE(account_set_id, account_code, currency)
);

-- 银行对账单（对应 yhdzd）
CREATE TABLE bank_statement (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id INTEGER NOT NULL,
  account_code TEXT NOT NULL, biz_date TEXT,
  debit REAL DEFAULT 0, credit REAL DEFAULT 0,
  settle_type TEXT, bill_no TEXT,
  matched INTEGER DEFAULT 0,         -- dzbz 对账标志
  match_batch INTEGER,               -- dzbh
  source TEXT DEFAULT 'import'       -- 区分手工/导入
);

-- 结算方式字典（对应 jslb）
CREATE TABLE settle_type (
  account_set_id INTEGER NOT NULL,
  code TEXT NOT NULL, name TEXT,
  PRIMARY KEY(account_set_id, code)
);
```

### 2.2 固定资产模块表

```sql
-- 固定资产卡片主表（对应 zc_gdzc，保留全部 ACD 字段）
CREATE TABLE fixed_asset (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id INTEGER NOT NULL,
  asset_no TEXT NOT NULL,            -- bh
  asset_name TEXT,                   -- mc
  category_code TEXT,                -- sblb
  status_code TEXT,                  -- sbzt
  dept_code TEXT,                    -- sydw 使用部门
  purpose_code TEXT,                 -- ytbh 用途（决定折旧费用科目）
  acquire_date TEXT,                 -- gzrq 购置日期
  start_depr_date TEXT,             -- syrq 起折日期
  original_value REAL DEFAULT 0,     -- yz 原值
  salvage_rate REAL DEFAULT 0,       -- czl 残值率
  salvage_value REAL DEFAULT 0,      -- yzcz 预计净残值
  depr_method TEXT,                  -- zjff 折旧方法(平均年限/工作量/双倍余额/年数总和)
  use_months INTEGER,                -- synx 使用月数
  use_years INTEGER,                 -- synx_y
  total_workload REAL,               -- zgzl 工作量法-总工作量
  depr_months_done INTEGER DEFAULT 0,-- ljzjy 已提月数
  workload_done REAL DEFAULT 0,      -- ljgzl 累计工作量
  accum_depr REAL DEFAULT 0,         -- ljzje 累计折旧
  net_value REAL DEFAULT 0,          -- zcjz 净值
  card_no TEXT,                      -- bjbh
  qty INTEGER DEFAULT 1,             -- sl
  unit TEXT,                         -- dw 计量单位
  user_name TEXT,                    -- syr 使用人
  keeper TEXT,                       -- bgr 保管人
  source TEXT,                       -- ly 来源
  install_place TEXT,                -- azdd
  is_foreign INTEGER DEFAULT 0, foreign_value REAL,  -- wbbz/wbje
  scrap_reason TEXT, scrap_date TEXT,-- zxyy/zxrq
  remark TEXT,                       -- bz
  acd_raw TEXT,                      -- 兼容兜底：原始整行 JSON，保证 100% 可逆
  created_at TEXT, updated_at TEXT,
  UNIQUE(account_set_id, asset_no)
);

-- 月折旧计提表（对应 zc_yzjb）
CREATE TABLE fixed_asset_depr (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id INTEGER NOT NULL,
  asset_no TEXT NOT NULL,
  year INTEGER, month INTEGER,
  dept_code TEXT, purpose_code TEXT, depr_method TEXT,
  month_depr REAL DEFAULT 0,         -- yzje 当月折旧额
  accum_depr REAL DEFAULT 0,         -- ljzje
  UNIQUE(account_set_id, asset_no, year, month)
);

-- 原值增减 / 变动流水（对应 zc_yzzj + zc_sbbd）
CREATE TABLE fixed_asset_change (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id INTEGER NOT NULL,
  asset_no TEXT NOT NULL,
  change_date TEXT, change_item TEXT,
  old_value REAL, new_value REAL,
  amount REAL, remark TEXT, operator TEXT
);

-- 字典：类别 / 状态 / 用途 / 部门 / 来源 / 注销原因
CREATE TABLE fixed_asset_category (account_set_id INTEGER, code TEXT, name TEXT,
  salvage_rate REAL, account_code TEXT, PRIMARY KEY(account_set_id, code));   -- zc_sblb
CREATE TABLE fixed_asset_status   (account_set_id INTEGER, code TEXT, name TEXT,
  depreciable INTEGER, PRIMARY KEY(account_set_id, code));                    -- zc_sbzt
CREATE TABLE fixed_asset_purpose  (account_set_id INTEGER, code TEXT, name TEXT,
  expense_account TEXT, PRIMARY KEY(account_set_id, code));                   -- zc_sbyt
CREATE TABLE fixed_asset_dept     (account_set_id INTEGER, code TEXT, name TEXT,
  PRIMARY KEY(account_set_id, code));                                         -- zc_sydw
```

> **可逆性保障**：主表保留 `acd_raw`（整行原始字段 JSON）；导出 `.acd` 时优先用结构化字段重组，缺字段以 `acd_raw` 补齐，确保「导入 → 导出」字段无损。

## 三、ACD 导入兼容方案

在现有 `server/src/scripts/importAcdToCurrentAccountSet.ts`（生产级解析器，已正确处理 GBK + 定长头）基础上扩展，**复用** `parseAcdTables` / `splitTableRows`，不重写解析层。

### 3.1 改造点
1. `parseAcdTables` 已读取全部 `rhsj\*.txt`，新增对 `cn_mx / cn_nc / yhdzd / jslb / zc_gdzc / zc_yzjb / zc_sblb / zc_sbzt / zc_sbyt / zc_sydw / zc_yzzj / zc_sbbd` 的取表。
2. 新增字段映射函数（按 §1 字段顺序与 `data_stru.txt` 列顺序逐列解析），分别落库到 §2 的表。
3. 复用现有事务 + 进度上报（`taskQueue` / `reportTaskManager`）。
4. 导入统计 `stats.importedTables` 增加上述表名，便于 `ACD导入完整性测试报告` 校验。

### 3.2 字段顺序锚定（关键，防错位）
解析必须严格按 `data_stru.txt` 的列定义顺序切分 `\t`，例如 `zc_gdzc` 共 50+ 列、`cn_mx` 共 28 列。计划在代码中以**常量数组**写死列名顺序（来源于本文件 §1），并对列数做断言校验，列数不符时记 warning 而非静默丢弃。

## 四、后端 API 设计

### 4.1 出纳
- `routes/cashierJournal.ts`：日记账 CRUD、按科目/月份/币别查询、余额结转。
- `routes/cashierReconcile.ts`：银行对账（自动匹配 `cashier_journal` ↔ `bank_statement`，按金额+日期+票据号）、对账单导入(Excel)、未达账项表。
- `services/cashierQuery.ts`：现金/银行日记账查询、日清月结、与凭证联查（通过 `voucher_*` 字段回链记账凭证）。
- 与凭证联动：录入凭证时若涉及现金/银行科目，可选自动生成 `cashier_journal`（参数 `gdzc_bz1` 类似开关）。

### 4.2 固定资产
- `routes/fixedAsset.ts`：卡片增删改查、批量导入、变动（增加/减少/部门转移）。
- `routes/fixedAssetDepr.ts`：计提折旧（按 §1 折旧方法）、生成折旧凭证、折旧结转月份控制（对应 `xt_zc_jzyf`）、折旧清单/汇总表。
- `services/fixedAssetDepreciation.ts`：四种折旧算法
  - 平均年限法 = (原值 − 净残值) / 使用月数
  - 工作量法 = (原值 − 净残值) / 总工作量 × 当月工作量
  - 双倍余额递减法
  - 年数总和法
- 字典维护路由：类别/状态/用途/部门（复用现有 base 模块风格）。

## 七、风险与注意事项

1. **字段错位**：ACD 文本为定长 `\t` 行，空字段也占位；必须按列数断言，禁止 `filter(Boolean)` 误删空列。`splitTableRows` 当前对**行**做 `filter(Boolean)`（删空行）是安全的，但对**列**不可过滤。
2. **GBK 编码**：沿用现有 `decodeTableContent`，勿用 utf8。
3. **折旧起算口径**：润衡按 `syrq`(起折日期) 次月计提；需与 `xt_zc_jzyf` 已结转月份对齐，避免重复计提。
4. **多账套隔离**：所有新表带 `account_set_id`，导入时绑定当前账套。
5. **二进制照片** `zc_pic`：单独存为 BLOB 或文件，导入可后置（M4 可选）。
6. **范围控制**：`pjnr1/pjnr2/rkdk0` 等进销存票据表本期仅结构兼容、不做业务 UI。
