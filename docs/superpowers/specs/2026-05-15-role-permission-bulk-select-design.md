# 角色权限批量选择设计

## 背景

角色管理页面的编辑弹窗当前按权限模块分组展示权限。每个模块标题已有一个复选框，可对该模块执行全选或取消全选，但缺少明确的“反选”操作，也缺少跨模块的一键全选和反选。

## 目标

在角色权限编辑区域增加批量选择能力：

- 权限配置区域顶部增加全局“全部全选”和“全部反选”。
- 每个权限分组标题右侧增加分组“全选”和“反选”。
- 保留现有模块复选框的全选、取消全选和半选状态。
- 不修改后端接口和数据结构。

## 设计

修改 `client/src/views/system/Role.vue`。

### 页面结构

权限配置加载完成后，在所有权限分组上方增加顶部工具栏：

- `全部全选`：选择所有模块下的所有权限。
- `全部反选`：对所有权限执行反向选择。

每个分组的 `perm-group-header` 调整为左右布局：

- 左侧保留模块复选框，继续显示模块名称和半选状态。
- 右侧增加两个文字按钮：`全选`、`反选`。

### 交互规则

- 分组全选：将当前分组内所有权限 code 加入 `form.permissions`，使用 `Set` 去重。
- 分组反选：当前分组内已选权限移除，未选权限加入。
- 全部全选：将所有分组权限 code 加入 `form.permissions`，使用 `Set` 去重。
- 全部反选：对所有分组权限 code 执行反选。
- 现有分组复选框不改变语义：勾选时全选该分组，取消勾选时清空该分组。

### 数据流

所有操作只更新前端表单字段 `form.permissions`。保存时仍沿用现有 `handleSave`，通过 `/system/roles` 或 `/system/roles/:id` 提交完整角色表单。

### 样式

新增少量 scoped 样式：

- 顶部工具栏使用浅色背景、边框和右对齐按钮，和现有权限分组风格保持一致。
- 分组标题使用 flex 布局，让模块复选框与分组操作按钮分列显示。

## 验证

- 运行前端类型检查，确认 `Role.vue` 无 TypeScript 错误。
- 手动验证角色编辑弹窗：
  - 顶部全部全选能选中所有权限。
  - 顶部全部反选能反转所有权限。
  - 分组全选只影响当前分组。
  - 分组反选只影响当前分组。
  - 原模块复选框半选状态正常。
