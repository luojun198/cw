# CW Finance 数据库设计文档

## 数据库概述

- **数据库类型**: SQLite
- **字符编码**: UTF-8
- **事务隔离级别**: SERIALIZABLE
- **WAL 模式**: 启用（Write-Ahead Logging）

## 核心表结构

### 1. 用户与账套

#### users (用户表)

```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  username TEXT NOT NULL UNIQUE,
  password TEXT NOT NULL,
  name TEXT,
  role TEXT DEFAULT 'user',
  is_enabled INTEGER DEFAULT 1,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  updated_at TEXT DEFAULT (datetime('now', 'localtime'))
);
```

**字段说明**:
- `id`: 用户 ID (UUID)
- `username`: 登录用户名（唯一）
- `password`: 密码（bcrypt 加密）
- `name`: 用户姓名
- `role`: 角色（admin/user）
- `is_enabled`: 是否启用（1=启用，0=禁用）

#### account_sets (账套表)

```sql
CREATE TABLE account_sets (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  code TEXT,
  start_date TEXT,
  end_date TEXT,
  is_enabled INTEGER DEFAULT 1,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  updated_at TEXT DEFAULT (datetime('now', 'localtime'))
);
```

**字段说明**:
- `id`: 账套 ID (UUID)
- `name`: 账套名称
- `code`: 账套编码
- `start_date`: 启用日期
- `end_date`: 停用日期

### 2. 会计科目

#### accounts (会计科目表)

```sql
CREATE TABLE accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_set_id TEXT NOT NULL,
  code TEXT NOT NULL,
  name TEXT NOT NULL,
  direction TEXT NOT NULL CHECK(direction IN ('debit', 'credit')),
  level INTEGER NOT NULL,
  parent_code TEXT,
  is_leaf INTEGER DEFAULT 1,
  is_cash INTEGER DEFAULT 0,
  is_bank INTEGER DEFAULT 0,
  is_enabled INTEGER DEFAULT 1,
  aux_category_ids TEXT,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  updated_at TEXT DEFAULT (datetime('now', 'localtime')),
  UNIQUE(account_set_id, code)
);

CREATE INDEX idx_accounts_code ON accounts(account_set_id, code);
CREATE INDEX idx_accounts_parent ON accounts(account_set_id, parent_code);
```

**字段说明**:
- `code`: 科目编码（如 1001、100101）
- `name`: 科目名称
- `direction`: 余额方向（debit=借方，credit=贷方）
- `level`: 科目级次（1-6）
- `parent_code`: 父科目编码
- `is_leaf`: 是否末级科目（1=是，0=否）
- `is_cash`: 是否现金科目
- `is_bank`: 是否银行科目
- `aux_category_ids`: 辅助核算类别 ID（JSON 数组）

**科目编码规则**:
- 一级科目: 4 位（如 1001）
- 二级科目: 6 位（如 100101）
- 三级科目: 8 位（如 10010101）
- 最多支持 6 级科目

### 3. 凭证系统

#### vouchers (凭证主表)

```sql
CREATE TABLE vouchers (
  id TEXT PRIMARY KEY,
  account_set_id TEXT NOT NULL,
  voucher_type_id TEXT NOT NULL,
  voucher_no TEXT NOT NULL,
  voucher_date TEXT NOT NULL,
  year INTEGER NOT NULL,
  period INTEGER NOT NULL,
  summary TEXT,
  status TEXT DEFAULT 'draft' CHECK(status IN ('draft', 'audited', 'posted')),
  maker_id TEXT,
  auditor_id TEXT,
  poster_id TEXT,
  audit_time TEXT,
  post_time TEXT,
  attachment_count INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  updated_at TEXT DEFAULT (datetime('now', 'localtime')),
  UNIQUE(account_set_id, voucher_type_id, voucher_no, year, period)
);

CREATE INDEX idx_vouchers_date ON vouchers(account_set_id, voucher_date);
CREATE INDEX idx_vouchers_status ON vouchers(account_set_id, status);
CREATE INDEX idx_vouchers_period ON vouchers(account_set_id, year, period);
```

**字段说明**:
- `voucher_type_id`: 凭证类型 ID
- `voucher_no`: 凭证号（如 记-001）
- `voucher_date`: 凭证日期
- `year`: 会计年度
- `period`: 会计期间（1-12）
- `status`: 状态（draft=草稿，audited=已审核，posted=已过账）
- `maker_id`: 制单人 ID
- `auditor_id`: 审核人 ID
- `poster_id`: 过账人 ID

**凭证状态流转**:
```
draft (草稿) → audited (已审核) → posted (已过账)
     ↓              ↓                  ↓
  可编辑/删除    可反审核          可反过账
```

#### voucher_entries (凭证分录表)

```sql
CREATE TABLE voucher_entries (
  id TEXT PRIMARY KEY,
  account_set_id TEXT NOT NULL,
  voucher_id TEXT NOT NULL,
  account_id INTEGER NOT NULL,
  account_code TEXT NOT NULL,
  account_name TEXT NOT NULL,
  summary TEXT,
  direction TEXT NOT NULL CHECK(direction IN ('debit', 'credit')),
  amount REAL NOT NULL,
  dept_id INTEGER,
  project_id INTEGER,
  person_id INTEGER,
  aux_data TEXT,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  FOREIGN KEY (voucher_id) REFERENCES vouchers(id) ON DELETE CASCADE
);

CREATE INDEX idx_voucher_entries_voucher ON voucher_entries(voucher_id);
CREATE INDEX idx_voucher_entries_account ON voucher_entries(account_set_id, account_id);
CREATE INDEX idx_voucher_entries_balance
  ON voucher_entries(account_set_id, account_id, direction);
```

**字段说明**:
- `direction`: 借贷方向（debit=借方，credit=贷方）
- `amount`: 金额
- `dept_id`: 部门 ID（辅助核算）
- `project_id`: 项目 ID（辅助核算）
- `person_id`: 人员 ID（辅助核算）
- `aux_data`: 辅助核算数据（JSON）

**借贷平衡约束**:
- 每张凭证的借方合计必须等于贷方合计
- 在应用层验证，数据库不强制约束

#### voucher_types (凭证类型表)

```sql
CREATE TABLE voucher_types (
  id TEXT PRIMARY KEY,
  account_set_id TEXT NOT NULL,
  name TEXT NOT NULL,
  code TEXT NOT NULL,
  prefix TEXT,
  is_enabled INTEGER DEFAULT 1,
  sort_order INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  UNIQUE(account_set_id, code)
);
```

**常见凭证类型**:
- 记账凭证（记）
- 收款凭证（收）
- 付款凭证（付）
- 转账凭证（转）
- 结转凭证（结）

### 4. 余额系统

#### init_balances (期初余额表)

```sql
CREATE TABLE init_balances (
  id TEXT PRIMARY KEY,
  account_set_id TEXT NOT NULL,
  account_id INTEGER NOT NULL,
  year INTEGER NOT NULL,
  init_balance REAL DEFAULT 0,
  aux_item_id TEXT DEFAULT '',
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  updated_at TEXT DEFAULT (datetime('now', 'localtime')),
  UNIQUE(account_set_id, account_id, year, aux_item_id)
);

CREATE INDEX idx_init_balances_lookup
  ON init_balances(account_set_id, account_id, year);
```

**字段说明**:
- `init_balance`: 期初余额
- `aux_item_id`: 辅助核算项 ID（复合键，如 dept:123|proj:456）

#### account_balances (科目余额表)

```sql
CREATE TABLE account_balances (
  id TEXT PRIMARY KEY,
  account_set_id TEXT NOT NULL,
  account_id INTEGER NOT NULL,
  year INTEGER NOT NULL,
  period INTEGER NOT NULL,
  current_debit REAL DEFAULT 0,
  current_credit REAL DEFAULT 0,
  aux_item_id TEXT DEFAULT '',
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  updated_at TEXT DEFAULT (datetime('now', 'localtime')),
  UNIQUE(account_set_id, account_id, year, period, aux_item_id)
);

CREATE INDEX idx_account_balances_period
  ON account_balances(account_set_id, year, period);
CREATE INDEX idx_account_balances_account
  ON account_balances(account_set_id, account_id, year);
```

**字段说明**:
- `current_debit`: 本期借方发生额
- `current_credit`: 本期贷方发生额
- `aux_item_id`: 辅助核算项 ID

**余额计算公式**:
```
借方科目期末余额 = 期初余额 + 本期借方发生额 - 本期贷方发生额
贷方科目期末余额 = 期初余额 + 本期贷方发生额 - 本期借方发生额
```

### 6. 会计期间

#### voucher_periods (会计期间表)

```sql
CREATE TABLE voucher_periods (
  id TEXT PRIMARY KEY,
  account_set_id TEXT NOT NULL,
  year INTEGER NOT NULL,
  period INTEGER NOT NULL,
  start_date TEXT NOT NULL,
  end_date TEXT NOT NULL,
  is_closed INTEGER DEFAULT 0,
  closed_by TEXT,
  closed_at TEXT,
  created_at TEXT DEFAULT (datetime('now', 'localtime')),
  UNIQUE(account_set_id, year, period)
);
```

**字段说明**:
- `is_closed`: 是否结账（1=已结账，0=未结账）
- `closed_by`: 结账人 ID
- `closed_at`: 结账时间

**期间管理规则**:
- 已结账期间不能录入凭证
- 必须按顺序结账（不能跳过期间）
- 反结账后才能修改凭证

## 数据完整性

### 外键约束

```sql
-- 凭证分录外键
FOREIGN KEY (voucher_id) REFERENCES vouchers(id) ON DELETE CASCADE

-- 级联删除：删除凭证时自动删除所有分录
```

### 唯一约束

```sql
-- 凭证号唯一性
UNIQUE(account_set_id, voucher_type_id, voucher_no, year, period)

-- 科目编码唯一性
UNIQUE(account_set_id, code)

-- 余额记录唯一性
UNIQUE(account_set_id, account_id, year, period, aux_item_id)
```

### 检查约束

```sql
-- 借贷方向检查
CHECK(direction IN ('debit', 'credit'))

-- 凭证状态检查
CHECK(status IN ('draft', 'audited', 'posted'))

-- 金额非负检查
CHECK(amount >= 0)
```

## 性能优化

### 索引策略

**高频查询索引**:
```sql
-- 凭证查询（按日期、状态）
CREATE INDEX idx_vouchers_date ON vouchers(account_set_id, voucher_date);
CREATE INDEX idx_vouchers_status ON vouchers(account_set_id, status);

-- 账簿查询（按科目、方向）
CREATE INDEX idx_voucher_entries_balance
  ON voucher_entries(account_set_id, account_id, direction);

-- 余额查询（按科目、年度）
CREATE INDEX idx_init_balances_lookup
  ON init_balances(account_set_id, account_id, year);

-- 日志查询（按时间）
CREATE INDEX idx_operation_logs_query
  ON operation_logs(account_set_id, created_at);
```

### 查询优化

1. **使用 CTE 减少子查询**
2. **批量查询消除 N+1 问题**
3. **条件性 JOIN 避免行膨胀**
4. **游标分页优化深翻页**

### WAL 模式

```sql
PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
PRAGMA cache_size = -64000;  -- 64MB
PRAGMA temp_store = MEMORY;
```

**优点**:
- 读写并发性能更好
- 写入性能提升
- 数据库文件更小

**定期 Checkpoint**:
```sql
PRAGMA wal_checkpoint(TRUNCATE);
```

## 数据备份

### 备份策略

1. **自动备份**
   - 每天凌晨 2 点自动备份
   - 保留最近 30 天备份

2. **手动备份**
   - 用户可随时手动备份
   - 备份文件命名：`backup_YYYYMMDD_HHMMSS.db`

3. **备份恢复**
   - 支持从备份文件恢复
   - 恢复前自动备份当前数据

### 备份实现

```typescript
// 备份
const backupPath = path.join(backupDir, `backup_${timestamp}.db`)
db.backup(backupPath)

// 恢复
const sourceDb = new Database(backupPath, { readonly: true })
sourceDb.backup(currentDbPath)
```

## 数据安全

### 敏感数据加密

- 用户密码使用 bcrypt 加密
- JWT token 使用 HS256 签名

### 数据隔离

- 所有数据按 `account_set_id` 隔离
- 用户只能访问授权的账套数据

### 审计日志

- 记录所有关键操作
- 包含操作人、时间、详情
- 日志不可删除或修改

## 扩展性设计

### 多账套支持

- 所有业务表包含 `account_set_id` 字段
- 数据按账套完全隔离
- 支持账套间数据导入导出

### 自定义字段

- 辅助核算项支持自定义字段
- 字段配置存储在 `aux_category_fields` 表
- 字段值存储在 `aux_items.field_values`（JSON）

### 报表模板

- 支持自定义报表模板
- 模板配置存储在 `report_templates` 表
- 使用 JSON 配置报表结构和计算公式
