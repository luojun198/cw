# CW Finance 架构设计文档

## 系统概述

CW Finance 是一个基于 Web 的财务记账系统，采用前后端分离架构，支持政府会计制度的凭证管理、账簿查询、报表生成等核心功能。

## 技术栈

### 前端
- **框架**: Vue 3 (Composition API)
- **构建工具**: Vite
- **UI 组件库**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router
- **HTTP 客户端**: Axios
- **语言**: TypeScript

### 后端
- **运行时**: Node.js
- **框架**: Express
- **数据库**: SQLite (better-sqlite3)
- **认证**: JWT (jsonwebtoken)
- **语言**: TypeScript

### 测试
- **单元测试**: Vitest
- **E2E 测试**: Playwright

#### 1. Routes Layer (路由层)

**职责**:
- 处理 HTTP 请求和响应
- 参数验证和类型转换
- 调用 Services 层执行业务逻辑
- 格式化响应数据

**文件位置**: `server/src/routes/`

**示例**:
```typescript
router.get('/vouchers', (req: AuthRequest, res) => {
  const filters = {
    accountSetId: req.accountSetId,
    page: Number(req.query.page) || 1,
    pageSize: Number(req.query.pageSize) || 50,
    // ... 其他参数
  }

  const query = buildVoucherListQuery(filters, db)
  const list = db.prepare(query.listSql).all(...query.listParams)

  res.json({ code: 0, data: list })
})
```

#### 2. Services Layer (业务逻辑层)

**职责**:
- 实现核心业务逻辑
- 构建复杂查询
- 数据转换和计算
- 业务规则验证

**文件位置**: `server/src/services/`

**核心服务**:
- `voucherQuery.ts` - 凭证查询逻辑
- `voucherPosting.ts` - 凭证过账逻辑
- `ledgerQuery.ts` - 账簿查询逻辑
- `reportBalance.ts` - 报表余额计算
- `autoTransfer.ts` - 自动结转逻辑
- `accountService.ts` - 科目服务

#### 3. Database Layer (数据访问层)

**职责**:
- 数据库连接管理
- SQL 执行
- 数据库迁移
- 事务管理

**文件位置**: `server/src/db/`

**核心文件**:
- `index.ts` - 数据库初始化和连接
- `migrations.ts` - 迁移执行器
- `migrationList.ts` - 迁移列表
- `schema.sql` - 数据库表结构

## 核心功能模块

### 1. 凭证系统

**功能**:
- 凭证录入、编辑、删除
- 凭证审核、反审核
- 凭证过账、反过账
- 批量操作
- 自动结转

**关键文件**:
- 前端: `client/src/views/voucher/Entry.vue`
- 后端: `server/src/routes/voucher.ts`
- 业务逻辑: `server/src/services/voucherPosting.ts`

**数据流**:
```
用户操作 → 前端表单 → API 请求 → 路由层验证
→ 业务逻辑层处理 → 数据库操作 → 响应返回
```

### 2. 账簿系统

**功能**:
- 总账查询
- 明细账查询
- 序时账查询
- 余额表查询
- 辅助核算账簿

**关键文件**:
- 前端: `client/src/views/ledger/*.vue`
- 后端: `server/src/routes/ledger.ts`
- 查询逻辑: `server/src/services/ledgerQuery.ts`

**查询优化**:
- 使用 CTE 减少子查询
- 批量查询消除 N+1 问题
- 条件性 JOIN 避免行膨胀
- 游标分页优化深翻页

### 3. 报表系统

**功能**:
- 资产负债表
- 利润表（收入费用表）
- 现金流量表
- 动态报表

**关键文件**:
- 前端: `client/src/views/report/*.vue`
- 后端: `server/src/routes/report*.ts`
- 余额计算: `server/src/services/reportBalance.ts`

**性能优化**:
- 批量查询余额（`getBatchBalances`）
- 批量查询发生额（`getBatchPeriodSums`）
- 一次查询替代多次查询

## 性能优化

### 查询优化

1. **批量查询消除 N+1**
   - 资产负债表: 100+ 次查询 → 2 次查询
   - 利润表: 30+ 次查询 → 2 次查询

2. **CTE 优化子查询**
   - 总账查询: 2400 次子查询 → 1 次 CTE 查询

3. **条件性 JOIN**
   - 凭证列表: 避免不必要的 5x 行膨胀

4. **游标分页**
   - 深翻页: OFFSET 扫描 → 游标过滤

### 并发控制

1. **凭证编号生成**
   - 使用 `BEGIN IMMEDIATE` 排它锁
   - 事务回滚机制

2. **过账操作**
   - 事务包装确保原子性
   - 乐观锁防止并发冲突

### 缓存策略

1. **内存缓存**
   - 批量查询结果缓存在 Map 中
   - 减少重复数据库查询

2. **操作日志异步化**
   - 内存队列缓存日志
   - 批量写入减少 I/O

## 安全设计

### 认证与授权

1. **JWT 认证**
   - 登录后颁发 JWT token
   - 每次请求验证 token
   - Token 过期自动刷新

2. **权限控制**
   - 基于角色的访问控制（RBAC）
   - 账套级别数据隔离

### 安全防护

1. **SQL 注入防护**
   - 参数化查询
   - 输入验证和过滤

2. **路径遍历防护**
   - 文件路径验证
   - 限制访问范围

3. **XSS 防护**
   - 输入转义
   - Content-Security-Policy
