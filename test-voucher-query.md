# 凭证查询多条件查询测试计划

## 修复的问题

### 1. 辅助核算项目多选参数处理错误

**问题**：
- 后端只接受字符串类型的辅助项目参数，忽略数组参数
- 类型定义不一致：`voucherQuery.ts` 定义为 `number[]`，但实际传递字符串

**修复**：
- 修改 `server/src/routes/voucher.ts` 第576-627行
- 支持字符串（逗号分隔）和数组两种格式
- 统一转换为 `number[]` 数组传递给查询构建器

## 已知问题和修复

### 问题1：辅助核算项目多选参数处理错误 ✅ 已修复

**修复内容**：
- 修改 `server/src/routes/voucher.ts` 第576-627行
- 支持字符串（逗号分隔）和数组两种格式
- 统一转换为 `number[]` 数组

**修复代码**：
```typescript
// 支持字符串（逗号分隔）或数组
if (typeof value === 'string' && value) {
  auxItemsInput[categoryId] = value
} else if (Array.isArray(value) && value.length > 0) {
  auxItemsInput[categoryId] = value
}

// 将字符串或数组统一转换为数组
if (typeof itemIds === 'string') {
  auxItems[code] = itemIds.split(',').map(id => id.trim())
} else if (Array.isArray(itemIds)) {
  auxItems[code] = itemIds
}
```
