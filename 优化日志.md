### 6. 期初余额功能全面改造（导入/模板下载/年度切换/锁定/批量保存）

**修改文件：**
- `server/src/scripts/migrationList.ts`（新增 v4 迁移）
- `server/src/routes/baseInitBalance.ts`（修改 GET/POST，新增 batch 和 can-edit 接口）
- `client/src/views/base/InitBalance.vue`（全面重写）

**具体说明：**

1. **数据库迁移 v4**：`init_balances` 表新增 `opening_debit`、`opening_credit`、`pre_book_debit`、`pre_book_credit` 四个字段，解决前端四列（年初借方/贷方、账前借方/贷方）与数据库字段不匹配的问题

2. **后端接口改动**：
   - `GET /init-balances`：SQL 查询增加返回 4 个新字段
   - `POST /init-balances`：接收并保存 4 个新字段
   - `POST /init-balances/batch`（新增）：批量保存接口，事务包裹，自动计算 `init_balance`/`init_debit`/`init_credit`（= 年初 + 账前）
   - `GET /init-balances/can-edit`（新增）：检查该年度是否有已审核/已过账凭证，有则返回 `canEdit: false`

3. **前端改造**：
   - **年度选择器**：el-select 下拉，近 10 年可选，切换年度自动刷新数据和锁定状态
   - **下载模板**：动态 `import('xlsx')`，生成 Excel 模板（科目编码/名称/方向/年初借贷方，中途开账时额外包含账前借贷方）
   - **Excel 导入**：el-upload 拖拽上传 → xlsx 解析 → 按科目编码匹配 → 预览表格（显示匹配状态）→ 确认后调用 batch 接口
   - **锁定逻辑**：页面加载和切换年度时调用 `can-edit` 接口，有已审核/已过账凭证时所有输入框 disabled，保存/导入按钮禁用，显示 el-alert 警告
   - **批量保存**：`saveAll` 从逐行循环 POST 改为一次调用 `/init-balances/batch`，性能大幅提升

**原因：**
- 前端有年初借方/贷方和账前借方/贷方四列，但数据库只有 `init_balance`/`init_debit`/`init_credit` 三个字段，保存实际无效
- 期初余额需要按年度管理（如 2025 年期初、2026 年期初），原来硬编码当前年份不可切换
- 缺少导入功能，大量科目需要逐个手动录入
- 缺少修改限制，已有凭证后修改期初会导致账务数据不一致
- 逐行保存性能差，改为批量事务保存

**影响范围：**
- 需要重启后端服务（已自动重启）
- 数据库自动迁移（v4），已有数据的新字段默认为 0

### 3. 凭证录入 "=" 快捷键自动平衡修复

**修改文件：**
- `client/src/components/voucher/VoucherEntryForm.vue`

**具体说明：**
- 修改 `onDebitKeydown` 函数（第 917-938 行）和 `onCreditKeydown` 函数（第 943-964 行）中 `=` 键的处理逻辑
- **旧逻辑（错误）**：在借方按 `=` 直接填入贷方合计（`totalCredit`），在贷方按 `=` 直接填入借方合计（`totalDebit`）。一借多贷场景下会导致金额严重不平衡
- **新逻辑（正确）**：先排除当前行的金额，计算其他行的同方向合计，再用对方合计减去该值得到差额。公式：
  - 借方按 `=`：`diff = totalCredit - (totalDebit - 当前行借方)`
  - 贷方按 `=`：`diff = totalDebit - (totalCredit - 当前行贷方)`
- 使用 `toFixed(2)` 避免浮点精度问题
- 差额 ≤ 0 时不填入（说明当前方向已经超出或平衡）

**原因：**
- 一借多贷场景举例：借方 1000，贷方已有 600，第三行贷方按 `=` 应填 400（差额），旧逻辑填 1000（借方合计），导致贷方合计 1600
- 一贷多借场景同理

### 27. 统一坐标体系：元素坐标改用 mm 单位，画布与打印所见即所得

**修改文件：**
- `client/src/types/print.ts`
- `client/src/components/print/TemplateDesigner.vue`
- `client/src/components/print/PrintPreview.vue`
- `create-standard-template.js`

**具体说明：**

1. **根因分析：**
   - 旧方案中元素 x/y/width/height/fontSize 用 px 存储，但画布用 mm 设置尺寸和 padding
   - PrintPreview 直接把 px 值设为 left/top（px 单位），但页面宽度用 mm，导致 px 与 mm 混用，位置完全错乱
   - 不同 DPI 设备上 px→mm 换算不一致

2. **坐标体系统一为 mm：**
   - FieldElement 的 x/y/width/height 全部改为 mm 单位
   - fontSize 改为 pt 单位（打印标准单位）
   - 设计器画布用 CSS mm 渲染（浏览器自动处理 DPI 换算）
   - PrintPreview 直接用 mm 单位定位元素，fontSize 用 pt 单位
   - vue-draggable-resizable 只接受 px，通过 mm2px()/px2mm() 转换

3. **关键换算常量：**
   - 1mm = 3.7795275591px（96 DPI 标准）
   - 1pt = 1.333px（屏幕渲染时）
   - 拖拽/调整回调返回 px 值，需用 px2mm() 换回 mm（保留1位小数）

4. **页边距设置 UI：**
   - 工具栏添加上/下/左/右边距输入框（mm 单位，0-50mm）
   - 修改画布 padding 响应边距变化

5. **标准模版更新：**
   - 所有坐标从 px 转换为 mm（例如 x:22→x:5.8≈6, y:106→y:28）
   - fontSize 从 px 改为 pt（22px→18pt, 12px→10pt, 8px→6pt）
   - rowHeight 从 px 改为 mm（28px→7mm, 30px→8mm）
   - 内容区：190mm × 170mm

6. **框选坐标修复：**
   - getCanvasOffsetMm() 返回 mm 坐标
   - selectionBoxStyle 用 mm2px() 转换为 px 显示
   - 框选匹配直接用 mm 坐标比较

**原因：**
- 画布上控件位置和实际打印位置不一致，根本原因是坐标单位不统一
- 需要添加页边距设置功能

**影响范围：**
- 旧模版数据（px 坐标）不兼容，需重新运行 create-standard-template.js 更新
- 所有元素坐标值含义变化（px→mm），属性面板步长和默认值调整

### 6. 修复凭证自动编号规则，排除结转凭证

**修改文件：**

**具体说明：**

修改 `getNextVoucherNo` 函数（第 228-284 行），在查询最大凭证编号时排除结转凭证：

```typescript
// 修改前：查询所有凭证的最大编号
SELECT MAX(...) as max_no
FROM vouchers
WHERE account_set_id=? AND year=? AND period=? 
  AND (voucher_type_id=? OR (voucher_type_id IS NULL AND ? IS NULL))

// 修改后：排除结转凭证
SELECT MAX(...) as max_no
FROM vouchers
WHERE account_set_id=? AND year=? AND period=? 
  AND (voucher_type_id=? OR (voucher_type_id IS NULL AND ? IS NULL))
  AND id NOT IN (SELECT voucher_id FROM auto_transfer_runs WHERE account_set_id=?)
```

**技术实现：**
- 通过 `auto_transfer_runs` 表识别结转凭证（该表记录了所有自动结转生成的凭证ID）
- 使用 `NOT IN` 子查询排除这些凭证
- 确保编号计算只基于手工录入的凭证

**原因：**
- 结转凭证是系统自动生成的，不应参与手工凭证的编号计算
- 例如：有"记账凭证1"、"结转凭证1"、"结转凭证2"时，新增凭证应为"记账凭证2"而非"结转凭证3"
- 避免手工凭证编号受结转凭证影响而跳号

**影响范围：**
