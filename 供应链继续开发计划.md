# 供应链模块 — 完整检查与继续开发计划

> 盘点日期：2026-06-20。基于当前代码（`server/src/routes/scmDoc.ts`、`scmBase.ts`、`services/scmCosting.ts`、`scmArAp.ts` 与 `client/src/views/scm/*`）。

## 一、现状总览（已具备，质量较好）

- **单据中台**：`scm_doc` + `scm_doc_line` + `scm_doc_type`（25 类单据）。草稿/审核/反审核/删除；下推链路（行级 `source_line_id` + 表头 `source_doc_id`）+ 上下游追溯页（可行内操作）+ 防重复下推 + 「已被下游引用」优雅拦截。
- **库存**：移动加权平均计价（`applyMovingAvgIn/Out`、`reverseDocMoves`），出入库/调拨(TR)/盘点(CK)/组装(AS)/拆卸(DS) 库存移动；现存量、收发存汇总、库存明细账；按仓库的负库存控制。
- **采购/销售全链路单据**：PQ 询价→PO 订单→PI 入库→RP 发票→PAY 付款；SQ 报价→SOa 订单→SO 发货→RS 发票→RCV 收款。
- **往来**：货物驱动（SO 发货→应收、PI 入库→应付，发票不参与）+ 收付款核销 + 往来对账/应收应付账龄 + 「往来管理」导航。
- **凭证联动**：`gen-voucher` 覆盖 采购/销售/生产/库存/组装拆卸 分录（收入、成本结转、进/销项税、应收/应付）。
- **缺料/MRP 雏形**：销售订单缺货分析（StockDispatch）→ 生成缺料单 MR → 按物料来源下推 采购订单/委外计划/生产计划。
- **基础档案**：物料（多级、主副单位、来源采购/委外/自制、供应商/委外厂、字段自定义、安全库存/提前期/采购员等）、往来单位（供应商/客户/委外厂）、仓库、单位、分类、BOM。
- **运维/体验**：模块初始化（清空 + 强制完整备份）、单据明细「实时库存」查询列、供应链参数（计价方式/负库存仓/对方科目/物料分级）。

## 二、关键缺口与问题（按优先级）

### P0 — 正确性（建议优先修）
1. **明细行扩展字段不落库**：`POST/PUT /scm/docs` 的 `scm_doc_line` INSERT 只存基础列 + `discount_rate/discount_amount`，**丢失** `price_with_tax / supplier_code / source_type / scrap_rate / process_fee / expire_date / produce_date / spec / unit / field_values / gift_flag / ref_no`。前端 DocForm 已采集并发送，保存即丢。（MR 因 `gen-downstream` 用 `COALESCE(行, 物料默认)` 掩盖，但行级覆盖全部失效。）
2. **计价方式只实现移动加权平均**：参数可选「全月平均/先进先出/后进先出/指定成本」，但 `scmCosting` 只实现 `moving_avg`，选其它会**静默按移动平均**计算；`scm_stock_batch` 从不写入 → 批次/FIFO/LIFO 不可用。

### P1 — 功能闭环缺失
3. **生产闭环缺失**：BOM→生产计划之后**没有执行**。无「计划→领料单(PL)/完工入库(PF)」路由；plan 的 `finished_qty/scrap_qty/yl_warehouse/fp_warehouse/bom_id` 多为空置。PL/PF/PS/PB/PJ 仅作通用出入库，与工单/BOM 无关联（无投入产出、在制、工单成本归集）。
4. **委外闭环缺失**：委外计划→委外发货(WO)/委外入库(WI) 无关联；WO/WI 仅通用出入库，无委外加工费、发料/收货对账、委外往来。
5. **生产/委外计划无操作**：计划只能新增/删除，无下达/执行/进度/关闭；追溯页只读。

### P2 — 增强 / 报表 / 体验
6. **报表偏少**：仅 收发存汇总 / 库存明细 / 往来对账 / 账龄。缺 采购明细与汇总、销售明细与汇总（毛利）、**库存预警**（`safety_stock` 已采集未用）、库龄/周转/ABC、采购建议（安全库存+提前期+在途）。
7. **批次/保质期**：`item.batch_flag` + `line.batch_no/expire_date` 采集但库存不按批次、无保质期预警。
8. **价格/政策未用**：`sale_price2/3`、`transfer_price`、`min_order_qty`、`lead_time_days` 已存档但未参与业务（多级售价、起订量校验、按提前期建议补货）。
9. **测试缺失**：SCM **无任何单测**（计价、过账、往来、下推超额校验等关键逻辑无回归保护）。
10. **采购资产联动**：采购入库生成资产卡片（`gen-assets`）的凭证联动标注「P5 完善」。

## 三、继续开发计划（分阶段）

> 进度：**阶段 0 ✅**（明细行落库、计价口径守卫、SCM 单测、PI 无供应商崩溃修复）；**阶段 1 ✅**（生产闭环：工单状态机 + 领料/完工 + 实际成本按计划数分摊·末批吸收，见 [生产闭环设计-阶段1.md](生产闭环设计-阶段1.md)）；**阶段 2 ✅**（委外闭环：委外发货/入库 + 料成本 + 加工费计入成品成本 + 委外厂应付 + 凭证拆分，复用工单框架）；**阶段 3 ✅**（库存预警 + 销售/采购报表含毛利 + 采购建议含在途，见 优化日志 2026-06-20）。**阶段 4** 按 1→5 顺序推进：**4-1 权限细化 ✅**（业务域+操作权限、补后端守卫、回填 v70）；**4-2 多级售价 ✅**（客户价格等级 v71、按单据类型取价、起订量软校验）；**4-3 SCM 打印模板 ✅**；**4-4 批次/保质期 ✅**（v72 批次流水、FIFO/手工扣批、临期预警、反审核回退；成本仍移动平均）。**4-5 指定成本计价 ⏸ 已搁置**（与既定成本口径相反，经用户确认）。**阶段 4 收尾，阶段 0~4 全部完成。** 收尾：**调拨/盘点批次维护 ✅**、**序列号管理 ✅**（v73，登记/核销/调拨换仓/查询，审核校验失败统一 400）；指定成本计价 ⏸ 仍搁置（口径冲突，经用户二次确认保持移动平均，不实施）。**供应链收尾完成，可选项仅余指定成本计价（已明确不做）。**

### 阶段 0：正确性补齐（P0）
- 修明细行扩展字段持久化：扩展 create/update 的 INSERT/UPDATE 列与 `GET /scm/docs/:id` 回读；回归下推/凭证/计价取数。
- 计价方式：落地「全月平均（期末一次加权）」+「先进先出（写 `scm_stock_batch`、出库按批扣减）」；未实现方式给出明确提示而非静默回退。
- 补 SCM 关键单测：移动平均出入库、SO 应收、PI 应付、下推超额拦截、反审核回滚。

### 阶段 1：生产闭环
- 生产计划状态机（草稿→下达→执行→完工/关闭）；BOM 展开生成领料建议。
- 「计划→领料单(PL)」「计划→完工入库(PF)」一键生成（带 `plan_id`，回写 `finished_qty/scrap_qty`）。
- 工单成本归集（领料成本 + 加工费 → 完工成本），完工入库按归集成本入库。

### 阶段 2：委外闭环
- 委外计划→委外发货(WO，发料/挂委外往来)→委外入库(WI，收货 + 加工费)。
- 委外加工费计入入库成本；委外往来（应付加工费）。

### 阶段 3：报表与预警
- 采购/销售 明细 + 汇总（含毛利）；库存预警（低于安全库存）；库龄/周转/ABC；单据综合查询；采购建议（安全库存 + 提前期 + 在途）。

### 阶段 4：进阶（按 1→5 顺序推进）
1. **权限细化** ✅（业务域 `scm:purchase/sale/inventory/production/outsource/arap/report` + 操作 `scm:audit/delete`；补全 production-plans/boms/arap 后端守卫；回填迁移 v70。见 优化日志 2026-06-20 阶段4-1）
2. **多级售价与价格政策** ✅（`scm_partner.price_level` v71；销售按客户等级取 `sale_price/2/3`、采购取进价；切换客户重新取价；起订量软校验。见 优化日志 2026-06-20 阶段4-2）
3. **SCM 单据打印模板** ✅（打印全页面 `/scm/docs/:id/print`：A4 版式+明细+金额大写+签字+`@media print`；查看页/列表入口。见 优化日志 2026-06-21 阶段4-3）
4. **批次/保质期** ✅（v72 批次流水表；FIFO/手工扣批、保质期到期、批次库存/临期预警、反审核回退；批次仅追溯、成本仍移动平均。序列号/调拨盘点批次/指定成本为后续。见 优化日志 2026-06-21 阶段4-4）
5. 指定成本计价 ⏸（已搁置）——与 4-4 锁定口径「批次仅追溯、成本仍移动平均」相反，经用户确认暂不实施。未来如需按批次实际成本结转再启。

## 四、建议起点
优先做 **阶段 0**（明细行落库 + 计价方式），因为它是正确性问题、影响已上线的采购/销售/计价/凭证，且改动集中、风险可控、可单测护航。随后按 生产闭环 → 委外闭环 → 报表 推进。
