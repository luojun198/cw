### 请求拦截器

`client/src/api/request.ts` 第 10-23 行：

```typescript
instance.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`  // ✅ 自动添加 token
    }
    const accountSetId = localStorage.getItem('accountSetId')
    if (accountSetId) {
      config.headers['X-AccountSet-Id'] = accountSetId  // ✅ 自动添加账套 ID
    }
    return config
  }
)
```

**结论：请求拦截器配置正确，会自动从 localStorage 读取 token。**

## 问题根因分析

根据以上验证，问题可能是以下几种情况之一：

### 1. 用户未登录或 token 失效 ⚠️

**症状：**
- 前端访问 `/report/run/1` 时，localStorage 中没有 token
- 或者 token 已过期（JWT 过期时间为 8 小时）

**验证方法：**
打开浏览器开发者工具（F12），在 Console 中执行：

```javascript
console.log('Token:', localStorage.getItem('token'))
console.log('AccountSetId:', localStorage.getItem('accountSetId'))
```

**如果输出为 `null`，说明用户未登录。**

**解决方案：**
1. 访问 http://localhost:5175/login
2. 使用 `admin` / `admin123` 登录
3. 登录成功后再访问 http://localhost:5175/report/run/1

---

### 2. 前端报错但未显示 ❌

**症状：**
- 报表执行过程中出现 JavaScript 错误
- 错误被捕获但未正确显示给用户

**验证方法：**
打开浏览器开发者工具（F12），查看 Console 标签页是否有红色错误信息。

**可能的错误：**
- `TypeError: Cannot read property 'xxx' of undefined`
- `Network Error`
- `Request failed with status code 401`

---

### 3. 报表数据为空但未提示 ⚠️

**症状：**
- 报表执行成功，但返回的数据为空
- 前端未正确处理空数据情况

**验证方法：**
打开浏览器开发者工具（F12），切换到 Network 标签页：
1. 刷新页面
2. 找到 `/api/report/templates/1/execute` 请求
3. 查看 Response 内容

**如果 Response 中 `status: 'completed'` 但 `result.sheets` 为空或数据为空，说明是数据问题。**

---

### 4. 前端渲染问题 🎨

**症状：**
- 报表数据获取成功
- 但前端渲染时出现问题（如样式错误、表格未显示）

**验证方法：**
打开浏览器开发者工具（F12），切换到 Elements 标签页，检查页面 DOM 结构。

---

## 推荐诊断步骤

### 第一步：检查登录状态

1. 打开 http://localhost:5175/report/run/1
2. 按 F12 打开开发者工具
3. 切换到 Console 标签页
4. 执行以下命令：

```javascript
console.log('Token:', localStorage.getItem('token'))
console.log('AccountSetId:', localStorage.getItem('accountSetId'))
console.log('UserInfo:', localStorage.getItem('userInfo'))
```

**如果 token 为 null，请先登录：**
- 访问 http://localhost:5175/login
- 使用 `admin` / `admin123` 登录

### 第二步：检查网络请求

1. 切换到 Network 标签页
2. 刷新页面（Ctrl+R 或 Cmd+R）
3. 找到以下请求：
   - `/api/report/templates/1/execute` - 执行报表
   - `/api/report/tasks/xxx` - 查询任务状态
4. 查看请求的 Status Code：
   - **200**：成功
   - **401**：未授权（需要登录）
   - **500**：服务器错误
5. 查看 Response 内容，确认数据是否正确返回

### 第三步：检查控制台错误

1. 切换到 Console 标签页
2. 查看是否有红色错误信息
3. 如果有错误，记录完整的错误堆栈
