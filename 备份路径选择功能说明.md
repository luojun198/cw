# 备份路径选择功能说明

## 功能概述

在手动备份时，用户可以选择将备份文件保存到不同的位置：
- **默认路径**：使用系统设置中配置的备份路径
- **自定义路径**：临时指定一个备份路径（仅本次有效）

## 使用方法

### 1. 访问备份页面
打开浏览器访问：http://localhost:5175/security/backup

### 2. 点击"立即备份"按钮
会弹出"选择备份路径"对话框

### 3. 选择备份方式

#### 方式A：使用默认路径
1. 选择"默认路径"单选按钮
2. 会显示当前系统配置的备份路径
3. 点击"确认备份"
4. 备份文件将保存到系统设置的路径

#### 方式B：使用自定义路径（手动输入）
1. 选择"自定义路径"单选按钮
2. 在输入框中输入备份路径
   - Windows 示例：`D:\backups` 或 `E:\移动硬盘\备份`
   - macOS 示例：`/Users/用户名/Documents/backups`
   - Linux 示例：`/home/用户名/backups`
3. 点击"确认备份"
4. 备份文件将保存到指定的自定义路径

#### 方式C：使用自定义路径（浏览器选择）
1. 选择"自定义路径"单选按钮
2. 点击输入框右侧的"浏览"按钮
3. 系统会打开文件夹选择器
4. 选择目标文件夹
5. 路径会自动填入输入框
6. 点击"确认备份"
7. 备份文件将保存到选择的文件夹

## 功能特点

### 1. 路径验证
- ✅ 自动检查路径是否存在
- ✅ 自动检查路径是否可写
- ✅ 如果路径不存在，会尝试自动创建
- ✅ 无效路径会显示详细的错误信息

### 2. 系统文件夹选择器
- ✅ Windows：使用 PowerShell 的 FolderBrowserDialog
- ✅ macOS：使用 osascript 的 choose folder
- ✅ Linux：使用 zenity（需要安装）

### 3. 备份信息提示
- ✅ 成功时显示文件名和大小
- ✅ 失败时显示具体错误原因

### 4. 灵活性
- ✅ 自定义路径仅本次有效，不影响系统设置
- ✅ 可以手动输入路径，也可以使用浏览器选择
- ✅ 支持复制粘贴路径

## 错误处理

### 常见错误及解决方法

1. **"请输入自定义备份路径"**
   - 原因：选择了自定义路径但未输入内容
   - 解决：输入路径或点击"浏览"按钮选择

2. **"自定义路径无效: 无法创建备份目录"**
   - 原因：路径不存在且无法创建（如盘符不存在）
   - 解决：检查路径是否正确，确保盘符存在

3. **"自定义路径无效: 备份路径不可写"**
   - 原因：没有写入权限
   - 解决：选择有写入权限的目录，或以管理员身份运行

4. **"自定义路径无效: 备份路径必须是一个目录，而不是文件"**
   - 原因：输入的是文件路径而不是文件夹路径
   - 解决：输入正确的文件夹路径

5. **"系统未安装文件选择器工具（zenity）"**（仅 Linux）
   - 原因：Linux 系统未安装 zenity
   - 解决：手动输入路径，或安装 zenity：`sudo apt install zenity`

## 使用场景

### 场景1：日常备份
- 使用默认路径
- 备份文件统一保存到配置的目录
- 方便管理和自动清理

### 场景2：重要备份到移动硬盘
- 使用自定义路径
- 选择移动硬盘的路径（如 `E:\财务备份`）
- 备份文件保存到移动硬盘，便于异地保存

### 场景3：临时备份到桌面
- 使用自定义路径
- 选择桌面路径（如 `C:\Users\用户名\Desktop`）
- 方便快速访问和传输

### 场景4：备份到网络共享文件夹
- 使用自定义路径
- 输入网络路径（如 `\\服务器\共享文件夹\备份`）
- 备份文件保存到网络存储

## 技术实现

### 前端
- 使用 Element Plus 的 Dialog 组件
- Radio 单选按钮选择路径类型
- Input 输入框支持手动输入和浏览器选择
- 实时验证和错误提示

### 后端
- 新增 `GET /api/security/backups/select-folder` API
- 根据操作系统调用相应的文件夹选择器
- 路径验证和权限检查
- 支持自定义路径参数的备份操作

## 注意事项

1. **自定义路径仅本次有效**
   - 不会修改系统设置中的备份路径
   - 下次备份仍需重新选择

2. **路径格式**
   - Windows：使用反斜杠 `\` 或正斜杠 `/`
   - macOS/Linux：使用正斜杠 `/`
   - 建议使用绝对路径

3. **权限要求**
   - 确保对目标文件夹有写入权限
   - Windows 可能需要管理员权限

4. **磁盘空间**
   - 确保目标磁盘有足够的空间
   - 备份文件大小通常在 10-50 MB
