# 用户体验改进计划（阶段二）

> 纯 Web 版行政事业单位财务记账系统 - 用户体验优化

## 改进目标

提升系统易用性、操作效率和用户满意度，降低学习成本，减少操作错误。

---

## 优先级分类

- **P0（高优先级）**：影响核心业务流程，用户强烈需求
- **P1（中优先级）**：提升操作效率，改善用户体验
- **P2（低优先级）**：锦上添花，长期优化

---

## 一、交互体验优化（P0）

### 1.1 加载状态与反馈优化
**问题**：用户不清楚系统是否在处理请求，容易重复点击

**改进方案**：
- [ ] 所有数据加载增加 Loading 状态（表格、表单、报表）
- [ ] 按钮点击后增加 loading 状态，防止重复提交
- [ ] 长时间操作（如备份恢复、批量操作）增加进度提示
- [ ] 统一错误提示样式和位置（右上角 Toast）
- [ ] 成功操作增加明确的反馈信息

**涉及页面**：全部页面

### 1.2 表单验证优化
**问题**：表单验证不够友好，错误提示不明确

**改进方案**：
- [ ] 凭证录入：实时验证借贷平衡，显示差额
- [ ] 科目编码：输入时实时检查编码规则和重复性
- [ ] 期初余额：实时验证借贷平衡
- [ ] 统一必填项标识（红色星号）
- [ ] 表单验证失败时自动聚焦到第一个错误字段

**涉及页面**：Entry.vue, Account.vue, InitBalance.vue

### 1.3 操作确认优化
**问题**：危险操作缺少二次确认，容易误操作

**改进方案**：
- [ ] 删除操作：显示删除对象的关键信息（凭证号、科目名称等）
- [ ] 批量操作：显示影响范围（如"将删除 15 条凭证"）
- [ ] 过账/反过账：显示操作影响（如"将影响 3 个科目余额"）
- [ ] 月结操作：显示月结期间和不可逆提示
- [ ] 统一确认对话框样式和文案

**涉及页面**：Audit.vue, Entry.vue, Account.vue

---

## 二、数据展示优化（P0）

### 2.1 表格体验优化
**问题**：表格数据展示不够直观，缺少快捷操作

**改进方案**：
- [ ] 凭证列表：增加状态标签（草稿/已审核/已过账）
- [ ] 科目列表：增加层级缩进显示，优化树形结构
- [ ] 金额显示：统一千分位格式，负数红色显示
- [ ] 表格排序：支持多列排序（如按日期+凭证号）
- [ ] 表格筛选：增加快捷筛选（本月、本年、未审核等）
- [ ] 空状态优化：无数据时显示友好提示和操作引导

**涉及页面**：Query.vue, Audit.vue, Account.vue, 所有账簿页面

### 2.2 报表展示优化
**问题**：报表数据密集，不易阅读

**改进方案**：
- [ ] 增加报表打印预览功能
- [ ] 支持报表数据导出（Excel、PDF）
- [ ] 关键指标高亮显示（如资产总计、本期盈余）
- [ ] 增加报表数据对比（同比、环比）
- [ ] 支持报表数据钻取（点击科目查看明细）

**涉及页面**：所有 report/ 下的页面

### 2.3 Dashboard 优化
**问题**：工作台信息不够丰富，缺少数据可视化

**改进方案**：
- [ ] 增加数据可视化图表（收入支出趋势、科目余额分布）
- [ ] 增加待办事项提醒（待审核凭证、未月结期间）
- [ ] 增加快捷操作入口（快速录入凭证、查看报表）
- [ ] 增加系统使用统计（本月凭证数、操作日志）
- [ ] 支持自定义 Dashboard 布局

**涉及页面**：Dashboard.vue

## 三、操作效率优化（P1）

### 3.1 快捷键支持
**问题**：财务人员需要频繁录入数据，鼠标操作效率低

**改进方案**：
- [ ] 凭证录入：支持 Tab 键切换字段，Enter 键保存
- [ ] 凭证录入：支持 Ctrl+S 保存，Ctrl+N 新增
- [ ] 凭证审核：支持空格键快速审核
- [ ] 全局快捷键：Ctrl+F 快速搜索，Esc 关闭对话框
- [ ] 增加快捷键帮助面板（按 ? 显示）

**涉及页面**：Entry.vue, Audit.vue

### 3.2 智能输入优化
**问题**：重复输入工作量大

**改进方案**：
- [ ] 凭证录入：支持复制上一条分录
- [ ] 凭证录入：常用摘要快速选择（下拉历史记录）
- [ ] 科目选择：支持编码快速定位（输入 1001 自动跳转）
- [ ] 日期选择：增加快捷选项（今天、本月第一天、本月最后一天）
- [ ] 金额输入：支持简写（如 1k = 1000，1w = 10000）

**涉及页面**：Entry.vue, Query.vue

### 3.3 批量操作优化
**问题**：批量操作不够灵活

**改进方案**：
- [ ] 凭证审核：支持全选、反选、按条件选择
- [ ] 凭证审核：支持批量过账（审核+过账一步完成）
- [ ] 科目管理：支持批量导入导出
- [ ] 期初余额：支持 Excel 批量导入
- [ ] 辅助核算：支持批量启用/停用

**涉及页面**：Audit.vue, Account.vue, InitBalance.vue

## 四、导航与布局优化（P1）

### 4.1 面包屑导航
**问题**：用户不清楚当前位置

**改进方案**：
- [ ] 所有页面增加面包屑导航
- [ ] 面包屑支持点击返回上级
- [ ] 显示当前页面标题和操作路径

**涉及页面**：Layout.vue（全局）

### 4.2 页面布局优化
**问题**：部分页面信息密度过高

**改进方案**：
- [ ] 凭证录入：优化表单布局，减少滚动
- [ ] 科目管理：左侧树形结构，右侧详情编辑
- [ ] 报表页面：筛选条件固定在顶部，避免滚动后找不到
- [ ] 统一页面间距和留白，提升视觉舒适度

**涉及页面**：Entry.vue, Account.vue, 所有 report/ 页面

### 4.3 响应式优化
**问题**：小屏幕下体验不佳

**改进方案**：
- [ ] 表格支持横向滚动，固定关键列
- [ ] 表单在小屏幕下自动调整为单列布局
- [ ] 侧边栏支持折叠，增加内容区域空间
- [ ] 适配平板设备（iPad）

**涉及页面**：全部页面

## 五、数据安全与容错（P1）

### 5.1 自动保存
**问题**：凭证录入过程中意外关闭页面导致数据丢失

**改进方案**：
- [ ] 凭证录入：自动保存草稿（每 30 秒）
- [ ] 凭证录入：页面关闭前提示未保存数据
- [ ] 支持从草稿恢复数据

**涉及页面**：Entry.vue

### 5.2 操作撤销
**问题**：误操作后无法恢复

**改进方案**：
- [ ] 删除操作：支持撤销（30 秒内）
- [ ] 批量操作：显示操作历史，支持回滚
- [ ] 关键操作记录到操作日志

**涉及页面**：Audit.vue, Entry.vue

### 5.3 数据校验增强
**问题**：数据异常时缺少友好提示

**改进方案**：
- [ ] 凭证录入：检测科目余额异常（如现金科目出现负数）
- [ ] 期初余额：检测借贷不平衡
- [ ] 月结前检查：未审核凭证、未过账凭证提醒
- [ ] 自动结转前检查：科目配置完整性

**涉及页面**：Entry.vue, InitBalance.vue, AutoTransfer.vue

## 六、帮助与引导（P2）

### 6.1 新手引导
**问题**：新用户不知道如何开始

**改进方案**：
- [ ] 首次登录显示引导流程（创建账套 → 设置科目 → 录入期初 → 录入凭证）
- [ ] 关键页面增加操作提示（如"点击新增按钮录入凭证"）
- [ ] 空状态页面增加操作引导

**涉及页面**：Dashboard.vue, 各功能页面

### 6.2 帮助文档
**问题**：用户遇到问题无处查询

**改进方案**：
- [ ] 每个页面增加帮助按钮（右上角 ? 图标）
- [ ] 点击帮助显示当前页面操作说明
- [ ] 增加常见问题 FAQ
- [ ] 增加操作视频教程链接

**涉及页面**：全部页面

### 6.3 字段说明
**问题**：专业术语不易理解

**改进方案**：
- [ ] 关键字段增加 Tooltip 说明（如"辅助核算"、"过账"）
- [ ] 表单字段增加示例（如科目编码格式）
- [ ] 错误提示增加解决方案建议

**涉及页面**：全部页面

## 七、性能优化（P2）

### 7.1 列表分页优化
**问题**：大数据量时加载慢

**改进方案**：
- [ ] 凭证列表：支持虚拟滚动（大数据量）
- [ ] 科目列表：懒加载子节点
- [ ] 操作日志：增加分页大小选择（20/50/100）
- [ ] 报表数据：支持按需加载

**涉及页面**：Query.vue, Account.vue, Log.vue

### 7.2 缓存优化
**问题**：重复请求相同数据

**改进方案**：
- [ ] 科目列表：缓存到 localStorage，定期更新
- [ ] 凭证类型：缓存到 Pinia store
- [ ] 辅助核算项目：缓存到 Pinia store
- [ ] 用户信息：缓存到 Pinia store

**涉及页面**：全局优化

## 验收标准

1. **加载速度**：所有页面首次加载 < 2 秒
2. **操作响应**：按钮点击响应 < 300ms
3. **错误率**：用户误操作率降低 50%
4. **学习成本**：新用户 30 分钟内完成基本操作
5. **用户满意度**：用户满意度 > 85%

---

## 注意事项

1. 所有改进需保持与现有功能兼容
2. 优先优化高频使用的功能（凭证录入、审核、查询）
3. 每个改进需编写测试用例
4. 改进后需进行用户测试和反馈收集
5. 保持代码质量，遵循现有架构规范
