# Windows Server 2012 零依赖部署 - 打包说明

## 方案原理

```
开发机 (Node 25)                    目标服务器 (零依赖)
┌─────────────────┐                ┌─────────────────────┐
│  构建            │                │                     │
│  ├─ Vite 构建前端 │                │  deploy-final/      │
│  ├─ esbuild 打包  │   复制整个文件夹 │  ├─ node/          │
│  └─ npm install   │ ──────────────→│  ├─ server/        │
│    (用 Node 20)   │                │  ├─ client/dist/   │
│                   │                │  ├─ data/          │
└─────────────────┘                │  ├─ uploads/       │
                                    │  └─ start.bat      │
                                    └─────────────────────┘
```

## 为什么不用 PKG

| 对比项 | PKG (单 exe) | Portable Node.js |
|--------|-------------|-----------------|
| 原生模块兼容 | 差（固定 Node 版本） | 好（版本可控） |
| better-sqlite3 | 需要手动匹配 ABI | 自动下载预编译 |
| 调试难度 | 高 | 低（直接看日志） |
| 包大小 | ~50MB | ~107MB |
| 维护成本 | 高 | 低 |

## 核心兼容性问题

better-sqlite3 是 C++ 原生模块，`.node` 二进制文件必须与 Node.js ABI 版本严格匹配：

| better-sqlite3 | Node.js 最低版本 | ABI |
|---------------|-----------------|-----|
| v12.x         | 20.x            | 115 |
| v11.x         | 18.x            | 108 |
| v9.x          | 16.x            | 93 |

**关键步骤**：必须用 portable Node.js 20 执行 `npm install`，这样 better-sqlite3 会自动下载对应 ABI 115 的预编译二进制。

## 打包步骤

### 自动打包（推荐）

在项目根目录 `d:\kf\cw` 下，双击运行：

```
打包配置\一键打包.bat
```

### 手动打包

```batch
:: 1. 构建前端
cd client
npx vite build
cd ..

:: 2. 构建后端 bundle
cd server
node scripts/buildBundle.mjs
cd ..

:: 3. 准备目录
mkdir deploy-final\server deploy-final\client deploy-final\data deploy-final\uploads

:: 4. 下载解压 Node.js 20.18.3
::    https://nodejs.org/dist/v20.18.3/node-v20.18.3-win-x64.zip
::    解压后重命名为 deploy-final\node\

:: 5. 用 portable Node.js 安装依赖（关键！）
cd deploy-final\server
copy ..\..\server\package.json .
set PATH=..\node;%PATH%
npm install --omit=dev
cd ..\..\..

:: 6. 复制文件
copy server\dist\bundle.cjs deploy-final\server\
copy server\schema.sql deploy-final\server\
xcopy client\dist deploy-final\client\dist\ /E /I /Y
```

## 部署包目录结构

```
deploy-final/
├── node/                           # Node.js 20.18.3 portable (82MB)
│   ├── node.exe
│   └── ...
├── server/
│   ├── bundle.cjs                  # 后端代码 (1.67MB)
│   ├── schema.sql                  # 数据库结构
│   └── node_modules/               # 生产依赖 (22MB)
│       └── better-sqlite3/
│           └── build/Release/
│               └── better_sqlite3.node  # 预编译二进制
├── client/
│   └── dist/                       # 前端静态文件 (2.4MB)
│       ├── index.html
│       └── assets/
├── data/                           # 数据库（运行时创建）
├── uploads/                        # 上传文件
└── start.bat                       # 启动脚本
```

## start.bat 注意事项

- 使用**纯英文**，避免 GBK/UTF-8 编码乱码
- Windows Server 2012 默认代码页 936 (GBK)
- `chcp 65001` 在某些系统上会导致 bat 文件内中文乱码
- 服务端 Node.js 日志中的中文不受影响（Node.js 内部 UTF-8）

## 版本配置

| 组件 | 版本 | 说明 |
|------|------|------|
| Node.js | 20.18.3 LTS | portable，支持 Win Server 2012+ |
| better-sqlite3 | 12.8.0 | 预编译二进制，支持 Node 20+ |
| esbuild target | node18 | 兼容 Node 18+ 语法 |
| Vite | 5.x | 前端构建 |

## 升级指南

如果需要更换 Node.js 版本：

1. 修改 `一键打包.bat` 中的 `NODE_VER` 变量
2. 检查 better-sqlite3 是否支持新版本
3. 重新运行打包脚本

如果需要更换 better-sqlite3 版本：

1. 修改 `server/package.json` 中的 `better-sqlite3` 版本
2. 确保与 Node.js 版本兼容（见上方兼容表）
3. 重新运行打包脚本
