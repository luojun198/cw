# 生产闭环设计（阶段1）— 成本口径已对齐

> 前置：[供应链继续开发计划.md](供应链继续开发计划.md) 阶段1。口径决策（2026-06-20）：
> **完工成本 = 实际成本·按计划数分摊（末批吸收差异）**；**阶段1只归集料成本**（加工费/制费留阶段2，含委外）。

## 一、模型总览
- **工单 = 生产计划（复用 `scm_production_plan`）**。状态机：`draft 草稿 → released 下达 → closed 关闭`（在产/完工进度由数据推断：已领料、`finished_qty`）。
- **领料单(PL，出库)**：按 BOM×计划数生成领料建议（可改），审核时子件按移动均价出库，借生产成本/贷存货。补料(PB)同向、退料(PJ)反向。
- **完工入库(PF，入库)**：成品入库，借库存商品/贷生产成本；单位成本见下。
- **在制(WIP)** = 生产成本科目按工单的「已领 − 已结转完工」，工单关闭时归零。
- 单据用新增 `scm_doc.plan_id` 关联工单，便于按工单归集与追溯。

## 二、成本口径（核心，已对齐）
设工单计划数 `plan_qty`，按 `plan_id` 汇总**已审核**单据：
- `已领成本 = Σ(PL/PB 出库金额) − Σ(PJ 退料入库金额)`
- `已结转完工金额 = Σ(已审核 PF 入库金额)`
- `WIP_当前 = 已领成本 − 已结转完工金额`
- **单位成本** `unitCost = 已领成本 ÷ plan_qty`

**完工入库 PF 审核时**：
- 常规批次：本次入库金额 = `本次完工数 × unitCost`，成品按该单位成本并入成品仓（移动均价）。
- **末批吸收**：当 `已完工数 + 本次完工数 ≥ plan_qty`（达成/超产）**或**工单被「关闭」时，本次入库金额 = **`WIP_当前` 全部余额**（清零在制），差异/损耗/领料多少全部由末批吸收 → 保证 WIP 归零、生产成本科目该工单余额为 0。
- **报废 `scrap_qty`**：仅记录数量、不单独入库，其成本由良品吸收（已含在分摊中）。

边界与校验：
- 完工时若 `已领成本 = 0` → 拦截「请先领料归集成本后再完工」（阶段1要求先领料）。
- 允许超产：`finished_qty` 可累加超过 `plan_qty`，单位成本仍按 `plan_qty` 分摊，余额末批吸收。
- 多工单领同物料：按 `plan_id` 各自归集，互不影响。
- 关闭工单时若 `WIP_当前 > 0` 且无后续完工：提示「在制余额 X 未结清」，允许「生成末批完工吸收」或强制关闭（余额转差异，记一条说明）。

## 三、数据变更
- 迁移：`scm_doc` 增 `plan_id TEXT`（PL/PF/PB/PJ 关联工单；其余单据为 null）。
- `scm_production_plan.status` 枚举扩展：`draft / released / closed`（已有 `finished_qty/scrap_qty/bom_id/yl_warehouse/fp_warehouse/dept_code` 复用）。
- 不新增 WIP 累计列——WIP/已领/已完工均按 `plan_id` 实时汇总，避免冗余不一致。

## 四、接口（阶段1）
- `POST /scm/production-plans/:id/release` 下达（draft→released，校验 bom_id/plan_qty）。
- `POST /scm/production-plans/:id/close` 关闭（released→closed，WIP 未清结时按上文提示）。
- `GET /scm/production-plans/:id` 详情：含 BOM 用料、已领明细、完工明细、`已领成本/已完工数/WIP/unitCost/进度`。
- `GET /scm/production-plans/:id/issue-suggestion` 领料建议：`BOM 用量×(剩余计划数)×(1+scrap_rate) − 已领`。
- `POST /scm/production-plans/:id/gen-issue` 生成领料单(PL) 草稿（带 `plan_id` + 建议行 + 原料仓）。
- `POST /scm/production-plans/:id/gen-finish` 生成完工入库(PF) 草稿（成品×完工数，带 `plan_id` + 成品仓）。
- 审核：复用 `/scm/docs/:id/audit`；**对带 `plan_id` 的 PF 增加「按工单 unitCost/末批吸收」结转逻辑**；PL/PB 按移动均价出库即可。

## 五、前端
- 生产计划列表页：加 状态标签 + 进度（finished/plan）+ 行操作（下达/生成领料/生成完工/关闭/查看工单）。
- 工单详情页（新）：头部（状态/计划数/BOM/原料仓/成品仓/已领成本/WIP/单位成本/进度）+ BOM用料表 + 已领料单列表 + 完工入库列表，关键单据可跳转。

## 六、凭证（复用现有）
- PL/PB 出库：借「生产成本(5001)」/贷「存货」。
- PF 入库：借「库存商品(存货)」/贷「生产成本(5001)」，金额=本次入库金额（按 unitCost/末批）。
- 全工单结清后，生产成本科目该工单借贷相抵为 0。

## 七、实施拆分（建议顺序）
1. 迁移 `scm_doc.plan_id` + 工单状态机（release/close）+ 详情聚合接口。
2. 领料建议 + `gen-issue`（生成 PL 草稿，带 plan_id）。
3. 完工 `gen-finish` + **PF 审核成本结转（unitCost/末批吸收）**（核心）。
4. 前端 计划列表操作 + 工单详情页。
5. 单测：领料归集、完工分摊、末批吸收 WIP 归零、超产、退料冲减。

## 八、验收
建工单→下达→生成并审核领料（WIP 归集）→生成并审核完工入库（按 unitCost 结转）→末批/关单 WIP 归零→生产成本科目该工单余额=0；超产、退料、未领料完工拦截 等边界通过。
